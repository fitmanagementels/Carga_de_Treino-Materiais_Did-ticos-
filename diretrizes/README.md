# Diretrizes

Esta pasta guarda orientacoes operacionais e documentos-diretriz do projeto.

Use `prompts/` apenas para prompts e comandos que serao colados em ferramentas externas.

## Regra de atualizacao

Quando uma diretriz, roteiro, formatacao ou prompt precisar de ajuste, atualizar o proprio documento existente em vez de criar um novo arquivo para cada alteracao.

Crie novo documento apenas quando houver uma nova funcao real no fluxo, por exemplo: uma diretriz de produto que ainda nao existe, um output externo bruto, uma auditoria nova ou um artefato final separado.

Evite arquivos de "versao corrigida", "versao final 2", "ajustes", "complemento" ou historico intermediario quando o conteudo deve apenas substituir ou melhorar a diretriz ativa.

## Arquivo ativo

- `notebooklm-fluxo-producao.md`: diretriz operacional para usar o NotebookLM com corpus amplo, outputs brutos e auditorias.
- `fontes-notebooklm-produtos.md`: define quais arquivos devem e nao devem ser usados como fontes no notebook de producao.
- `criterios-linguagem-densidade-operacional.md`: define tom, densidade, nivel tecnico e grau de operacionalidade dos produtos finais.
- `tema-visual-xsteam.md`: define identidade visual, paleta, tipografia, geometria, imagens e composicao para slides e materiais didaticos.
- `diretriz-apresentacao-slides.md`: define o contrato de producao dos slides, incluindo campos obrigatorios por slide, estrutura visual, continuidade entre lotes e criterios de qualidade.
- `roteiro-apresentacao.md`: roteiro ativo slide a slide para producao da apresentacao no NotebookLM Studio.
- `diretriz-ebook-mobile-first.md`: define funcao, estrutura, densidade, capitulos e criterios de qualidade do ebook mobile-first.
- `roteiro-ebook-mobile-first.md`: roteiro ativo capitulo a capitulo para producao do ebook mobile-first no NotebookLM.
